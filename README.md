# CS 230 Portfolio Artifact

## The Gaming Room - Draw It or Lose It

1. Briefly summarize The Gaming Room client and their software requirements.

The Gaming Room wanted to expand their game, Draw It or Lose It, to support multiple platforms and users. They needed a design that allowed only one instance of the game to run while supporting multiple teams and players. The software also needed to be secure, scalable and capable of operating in distributed environments.

2. What did you do particularly well in developing this documentation?

I think I did a good job explaining technical concepts in a way that both developers and clients could understand. I also provided clear recommendations for operating systems, storage, memory management and security while keeping the client's requirements in mind throughout the project.

3. What about the process of working through a design document did you find helpful when developing the code?

Creating the design document helped me organize my thoughts before implementation. Planning the architecture, requirements and constraints early made it easier to understand how the different components would work together and reduced confusion during development.

4. If you could revise one part of your work, what would you improve?

If I could revise one section, I would expand the distributed systems portion. I would include more details about cloud deployment, fault tolerance and ways to maintain availability during outages as the application scales.

5. How did you interpret the user's needs and implement them into your software design?

I focused on the client's need for scalability, security and cross-platform compatibility. By recommending a Linux server environment and a client-server architecture, I designed a solution that supports multiple users while maintaining performance and protecting user data. Understanding the user's needs is important because software should solve real problems rather than simply implement technical features.

6. How did you approach designing software, and what techniques would you use in the future?

I approached the design process by gathering requirements first, analyzing possible solutions and then selecting the technologies that best matched the client's goals. In future projects, I would continue using design documents, UML diagrams and iterative planning to organize ideas and identify potential issues before development begins.
